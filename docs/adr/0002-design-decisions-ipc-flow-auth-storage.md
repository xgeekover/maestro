# ADR-0002 — IPC · 플로우 토폴로지 · 인증 · 저장소 진화

- 상태: **승인(Accepted)** — Phase 0 게이트에서 사용자 확정 (2026-06-28)
- 맥락: [docs/00-analysis.md](../00-analysis.md) §7 미해결 결정 O-1·O-2·O-8·O-9
- 선행: [ADR-0001](0001-phase0-foundational-decisions.md) (D1~D5)

## 결정 (Decisions)

### O-2. backend↔runner IPC — **gRPC**
- **결정**: 제어·텔레메트리 채널을 gRPC로 구현. 메시지 스키마는 `protocol/` 모듈의 `.proto`로 정의해 backend·runner가 공유.
- **근거**: 스키마 우선·양방향 스트리밍(상태/로그/메트릭 푸시 + 명령 수신)·성능·언어중립 코드생성. 수백 프로세스 텔레메트리에 적합.
- **결과**: `protocol/`이 gRPC 서비스/메시지의 단일 출처. 빌드에 protobuf 코드생성 플러그인 추가. 채널은 로컬 전용 바인딩 + 프로세스 핸드셰이크 토큰(T-9 완화).
- **대안**: 길이구분 JSON(단순하나 스키마/스트리밍 수작업), WebSocket(로컬 IPC 이점 적음) → 기각.

### O-9. 플로우 토폴로지 — **DAG 강제(사이클 금지)**
- **결정**: 플로우 그래프는 방향성 비순환 그래프(DAG)만 허용. 노드별 바운디드 큐 + 드롭/블록 백프레셔 정책.
- **근거**: 단순·예측 가능·백프레셔 용이. 무한 메시지 순환 위험 제거. node-RED식 MVP에 충분.
- **결과**: 플로우 저장/실행 시 사이클 검출·거부. 데이터 모델에 위상 정렬 기반 라우팅. 피드백 루프가 필요해지면 ADR로 사이클+루프가드 도입.

### O-8. 인증 — **로컬 계정(BCrypt) + JWT/세션**
- **결정**: 사용자/역할을 H2에 저장, BCrypt 해시. JWT(또는 세션)로 인증, 리소스 소유권 기반 인가.
- **근거**: 외부 IdP 의존 없이 자체 완결. D4(다중 사용자) 충족. 초기 구성·테스트 단순.
- **결과**: Spring Security 구성. 스크립트/플로우/모듈에 소유자·권한 부여(T-8 완화). SSO/OIDC는 후순위 확장.

### O-1. 저장소 진화 — **H2 서버모드 시작 + Postgres 전환경로**
- **결정**: H2 서버모드로 시작하되, 데이터 접근을 Spring Data로 추상화하고 표준 SQL·Flyway 마이그레이션을 유지.
- **근거**: 외부 의존 0으로 시작(D3) + D4(다중 사용자) 동시 쓰기 부하/HA 필요 시 저비용 Postgres 전환 확보.
- **결과**: 벤더 특화 SQL 회피, Flyway로 스키마 버전관리. 전환 트리거(동시 쓰기 병목·HA 요구) 도달 시 ADR로 Postgres 채택.
- **대안**: 지금부터 Postgres(D3 번복·외부의존), H2 파일모드 단순유지(다중쓰기 한계·전환 재작업) → 기각.
